## Bilingual structure

The site supports English and Simplified Chinese. Astro's built-in i18n routing maps each locale to a URL prefix.

| Locale | Path prefix | Example |
| ------ | ----------- | ------- |
| English (default) | `/en/` | `/en/` |
| Chinese | `/zh/` | `/zh/` |

The root `/` redirects the visitor to their preferred locale. Both locales share the same page components; language-specific copy is pulled from the `fragments` content collection.

## Content model

The site uses a single Astro content collection:

```
content/
└── fragments/
    └── *.md      # Localized text fragments
```

Fragments hold short pieces of copy (headings, labels, bio text) that differ between locales. They are loaded with Astro's `glob` loader and consumed by page components at build time.

<Note>
  The `fragments` collection has no enforced schema beyond the Markdown body. Frontmatter keys are defined informally and read by page components directly.
</Note>

## Local development

<Steps>
  <Step title="Install dependencies">
    From the repository root:

    ```bash
    pnpm install
    ```
  </Step>
  <Step title="Start the dev server">
    ```bash
    pnpm dev:www
    ```

    The site is available at `http://localhost:3001`.
  </Step>
</Steps>

## Building for production

```bash
pnpm build:www
```

Astro outputs the site to `websites/www/dist/`. Most pages are pre-rendered as static HTML; the root `/` redirect is rendered on demand (`export const prerender = false`). Vercel picks this up automatically on deploy.
